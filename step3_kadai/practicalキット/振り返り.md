# ✅ SQLite での直接操作とデータ削除まとめ

## 1. 直接DBの中身を削除する場合の方法
- SQLite3コマンドを使ってデータベースを直接操作する
- ターミナルで SQLite を起動する例：

  ```bash
  sqlite3 /path/to/CRM.db


注意点：

正しいデータベースファイルを開くこと（FastAPI などで使用している DB と同じ）

相対パスやルートが異なると「テーブルがない」などのエラーが発生する

対象のパスの位置までcdで移動し、
sqlite3 CRM.db   &
DELETE FROM customers WHERE customer_id = '';
をすることで解決。

2. SQLite の終了方法

SQLite の対話モードを終了して通常のターミナルに戻る方法：

.quit


または

.exit


キーボードショートカット：

Ctrl + D

3. テーブルのカラム構造の確認方法

カラム情報を確認するコマンド：

PRAGMA table_info(customers);


今回の customers テーブルの結果：

index	name	type	notnull	default	pk
0	customer_id	VARCHAR	1		1
1	customer_name	VARCHAR	1		0
2	age	INTEGER	1		0
3	gender	VARCHAR	1		0

customer_id は NOT NULL 制約 があるため、NULL は入らない

4. データ削除時の注意点

当初「customer_id が NULL のレコードがある」と思っていたが、実際には 空文字 "" が入っていた

NULL と空文字は SQL 上で明確に異なる

5. 空文字レコードの削除方法

空文字を持つレコードを削除する SQL：

DELETE FROM customers WHERE customer_id = '';


NULL の場合は別コマンド：

DELETE FROM customers WHERE customer_id IS NULL;

6. 今回の学び・注意点まとめ

✅ SQLite は 大文字小文字を区別（テーブル名に注意）

✅ 複数の CRM.db が存在する場合、どれを開いているか .tables で確認する

✅ NOT NULL 制約がある場合、NULL ではなく 空文字 が入っている可能性がある

✅ 空文字と NULL は SQL での扱いが異なる

✅ SQLite の終了は .quit または Ctrl + D で戻る
